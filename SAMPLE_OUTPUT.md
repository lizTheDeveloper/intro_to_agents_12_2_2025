# Sample Output

This document shows what the generated meeting notes look like.

## Example Generated Note

Here's what gets created in `meeting_notes/2024-12-03_Product_Planning_Q1_Roadmap.md`:

---

# Product Planning - Q1 Roadmap

**Date:** 2024-12-03
**Time:** 14:30

## Attendees
- Alice (Meeting Lead)
- Bob (Engineering)
- Charlie (Product)

## Summary
The team discussed the Q1 product roadmap with a focus on the mobile app redesign based on user feedback. Key decisions were made regarding priorities, and action items were assigned with specific deadlines. The team agreed to schedule a follow-up meeting to discuss technical implementation details.

## Action Items
- [ ] Charlie: Gather all user feedback and create summary document by Friday (Dec 6)
- [ ] Alice: Review current design mockups and provide feedback by next Tuesday (Dec 10)
- [ ] Bob: Send calendar invite for follow-up technical meeting (today)
- [ ] Charlie: Update project timeline in Notion by Friday (Dec 6)

## Full Transcript
```
[Meeting Start]

Alice: Good morning everyone! Thanks for joining today's product planning meeting.
Let's go over our Q1 roadmap.

Bob: Hi Alice, hi everyone. I think we should prioritize the mobile app redesign.

```

## Benefits of This Format

1. **Human Readable**: Standard Markdown that anyone can read
2. **Version Control Friendly**: Works great with Git
3. **Checkbox Compatible**: GitHub, VS Code, and many editors render checkboxes
4. **Search Friendly**: Easy to grep/search for specific items
5. **Export Ready**: Can convert to PDF, HTML, or other formats
6. **Organized**: ISO date format keeps files sorted chronologically

## Real-World Usage

### Before a Sprint Planning

```bash
python meeting_notes_agent.py
> todos
```

See all pending items before planning new work.

### After Completing Tasks

```bash
> Mark "Update documentation" as complete
> Mark "Review PR #123" as complete
```

Keep your notes up to date as you work.

### Weekly Review

```bash
> Show me all to-dos from this week
```

Review what was discussed and decided during the week.

### Sharing with Team

Simply share the Markdown files:
- Email them
- Check into Git
- Share in Slack/Teams
- Add to project wiki

## Customization Ideas

You can extend the format by modifying `save_meeting_notes()`:

- Add tags: `#engineering #design`
- Add priority: `- [ ] [HIGH] Task...`
- Add estimates: `- [ ] Task (2h)...`
- Add links: `- [ ] Review [PR #123](url)...`
- Add emoji: `- [ ] ðŸ”¥ Urgent task...`

The Markdown format is flexible and easy to customize!
